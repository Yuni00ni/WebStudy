HTTP 메서드(요청 방식) : get/post(form 태그 사용시)
- get : 요청 데이터를 url 뒤에 포함(패킷의 header -> 노출) / 검색(빠름)
- post : 패킷의 body에 포함(노출 x, 일반적으로는 데이터를 암호화시킴) / (로그인 / 회원가입 등에 사용)

redirect : 클라이언트에게 ~~ 주소로 다시 요청하라고 응답하는 방식
  => 클라이언트가 새로운 주소로 직접 요청하기 때문에 주소창의 주소도 해당 주소로 변경

sendFile : 파일을 응답하지만 주소는 바뀌지 않음(최초로 요청했던 주소만 그대로 노출)

db 연동
connection -> sql 실행 -> 결과처리 -> **connection 해제**
  => 해제하지 않으면 리소스가 과도하게 쓰이기 때문에 db연결에 문제가 생길 수 있음(물론 돈도 그만큼 많이 빠져나감)
